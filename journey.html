Internship Journey

Stage by Stage

This is not a highlight reel. It is what actually happened.

Stage 0
✓ Pass

The First Filter

Before anything started, they filtered us. This stage was about proving you were serious — analytical thinking, attention to detail, following instructions exactly. I made it through. That was the first sign that QA might actually be the right path for me.

Stage 1
✓ 75.8 — Pass

Functional Foundations

My first real QA task. I had to write test scenarios, test cases, and bug reports for a user registration feature. I passed, but the feedback was honest — my test steps were too vague, my format was not up to standard, and one of my scenarios was outside the scope entirely. I thought I was doing well. The score humbled me. First lesson: documentation is not just writing things down. It is writing things down so clearly that anyone can reproduce exactly what you did.

Stage 2
✗ 40 — Fail

API Testing & The Postman Mistake

My lowest score in the entire program. We were testing REST APIs using Postman and I submitted a JSON file instead of my Postman collection link. I did not even know the difference at the time. I scored 40. A mentor named Shehasnoname gave us a second chance and I used it. That stage taught me the most basic QA lesson — read the instructions. Twice. Then read them again.

Stage 3
✗ 59.33 — Fail ⚡ Premium

Automation & Hard Decisions

My code worked. I ran it myself, I knew it did. But on the day of the presentation, Git Bash would not run. I could not demonstrate what I had built. To make it worse, the code ran fine right after the presentation ended. On top of that, I had hardcoded values in my repo instead of using dynamic test data. Two problems at once. I failed the stage. I was not ready to leave so I paid for premium to stay in.

Stage 4
✗ 44.5 — Fail ⚡ Premium

CI/CD & The AI Lesson

The feedback was blunt and I deserved it. My tests were AI-generated and I had not taken the time to understand them myself. I had not applied the corrections from Stage 3. The mentor said I needed to go back to QA basics. They were right. I was still on premium. But instead of just surviving, I changed my approach completely. No more copying without understanding. From this point on, everything I submitted came from me actually doing the work.

Stage 5
✓ Pass

Real Team. Real Project. ClinSight.

This was the first stage where I worked inside a real cross-functional team. For the ClinSight MVP, I built a full QA test plan covering authentication, OTP verification, API validation, and session handling — complete with entry criteria, exit criteria, risk assessment, and a structured testing strategy. I was assigned actual tickets, wrote test cases, executed them, and logged defects. It was the closest thing to a real job I had experienced.

Stage 6
✓ 61 — Pass

Exploratory Testing & A Formatting Wake-Up Call

I passed but the feedback stung. The mentor said my document formatting was poor and that half of QA work is how you present your findings. If it is not properly formatted, it is not good enough — full stop. I had been so focused on finding bugs that I had been neglecting how I communicated them. That feedback changed how I approach every document I touch.

Stage 7
✓ 60 — Pass

Accessibility & API Security

I submitted 24 accessibility bugs — varied, not duplicates. My best catch was a keyboard tabbing issue where pressing past the Next button could trigger a violation. The mentor called it an excellent find because it combined accessibility, user fairness, and product logic all at once. I also mapped 12 API security issues to OWASP Top 10 categories and built a risk assessment that traced a full attack chain from fake employer registration to possible candidate data exposure.

Stage 8
✓ 65 — Pass

Performance Testing & My Strongest Work

This was my best stage. I ran load, soak, stress, and spike tests using k6. I used Lighthouse for frontend performance analysis. Midway through, I discovered that my original tests had been measuring the wrong path — invalid credentials that returned fast 401s without doing any real login work. I stopped, acknowledged the flaw, re-tested with real authenticated credentials, and corrected my findings. At just 20 users the real authenticated flow hit a 5.2s p95. At 150 users it collapsed entirely. After everything — the failures, the premium payments, the honest feedback — finishing strong here meant everything to me.
